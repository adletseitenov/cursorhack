# Поводырь v3 — voice-first: авто-описание, tap-to-talk, голосовые команды

Дата: 2026-07-08, хакатон Cursor Physical AI.

## Зачем

Незрячий пользователь не должен искать кнопки. Сценарий v3: повесил телефон,
нажал одну кнопку — дальше только голос и касание экрана в любом месте.

## Что изменилось относительно v2

### Авто-описание после старта камеры
«Старт камеры» сразу запускает цикл в режиме описания (`startNav(false)`),
голос объясняет: «Описываю, что впереди. Коснитесь экрана и скажите, куда вести».

### Tap-to-talk
Клик в любом месте страницы (кроме кнопок, поля цели и judge-зоны — их оставляем
зрячим для отладки) = нажатие «🎤». Защита: `listening`-гард против двойного
`rec.start()` (InvalidStateError).

### Голосовые команды (parseVoiceCommand)
- «стоп/стой/остановись/хватит/замолчи/пауза» → stopNav. ВАЖНО: в JS-регэкспах
  `\b` не работает с кириллицей — используем `(\s|$)`.
- «что вокруг/что впереди/опиши…» → цель очищается, режим описания.
- «(я) (хочу) дойти/подойти/веди (меня) (до|к) X» → цель = X (извлекается из фразы);
  не распозналось — вся фраза целиком становится целью (VLM понимает свободную форму).

### Прибытие → снова описание
`arrived` (2 тика подряд) больше не останавливает цикл: цель очищается, «Цель
достигнута! Описываю, что впереди» — контур живёт, пока не скажут «стоп».

### Темп описания
`DESCRIBE_TICK_MS = 1200` (навигация — по-прежнему 300): непрерывная болтовня
утомляет, описание реже.

### Первый тик через scheduleNext(300)
`startNav` больше не зовёт `tick()` синхронно: вступительная фраза успевает
начаться, и `scheduleNext` дождётся её конца — ответ VLM не обрывает речь.

## Промпты — победители шутаута (2026-07-08)

Метод: 3 варианта describe-промпта и 2 nav-промпта × 4 синтетических кадра
(стул в коридоре, проём слева, бутылка вплотную, лестница вниз) через боевой
прокси; 3 судьи-агента с линзами «безопасность», «голосовой UX», «формат».

Победил вариант B (оба): единственный дал `stop`+`hazard=true` на лестнице вниз
(проигравшие командовали «вперёд осторожно» — критический провал), даёт сторону
обхода препятствия и дистанции в шагах. В финальную версию добавлены фиксы
судейских замечаний: «ровно четыре поля» (nav B сыпал лишние поля в JSON),
«если stop — hazard=true» (консистентность), «не уверен в стороне — не выдумывай»
(desc B галлюцинировал «дверь справа»).

Ключевые свойства промптов:
- приоритет: опасность → препятствие+сторона обхода → путь/цель;
- дистанции в шагах (1 шаг ≈ 0.7 м);
- запрещены «изображение/кадр/вижу» — речь о мире, не о картинке;
- nav: цель не видна → команда поворота для поиска;
- arrived строго: цель перед грудью, большая часть обзора, меньше шага.

## Вне скоупа v3

Непрерывное распознавание речи (Android дребезжит нотификациями на каждый
рестарт SpeechRecognition, микрофон ловит TTS), TalkBack/VoiceOver-аудит,
чтение табличек, офлайн.
