# Поводырь v2 — прокси ключа, голосовая цель, режим описания

Дата: 2026-07-08, хакатон Cursor Physical AI. Утверждено Адлетом.

## Зачем

1. Ключ OpenRouter нельзя держать в браузере/git: OpenRouter отключает засвеченные
   ключи, публичный репо обязателен для сабмита.
2. Незрячий пользователь не может печатать цель в текстовое поле — цель задаётся голосом.
3. Пока цель не задана, приложение должно быть полезным: описывать, что впереди.

## Архитектура

### Бэкенд: Netlify Function `netlify/functions/vlm.mjs`
- POST `/api/vlm`, тело: `{model, messages, response_format?, max_tokens?}`.
- Функция добавляет `Authorization: Bearer ${process.env.OPENROUTER_API_KEY}`,
  форвардит на `openrouter.ai/api/v1/chat/completions`, возвращает статус и тело как есть.
- Белый список полей тела (не даём гонять произвольные запросы через наш ключ),
  max_tokens ≤ 300, таймаут апстрима 9 с (лимит функции на Free — 10 с).
- Ключ задаётся `netlify env:set OPENROUTER_API_KEY <key>` + редеплой. В git ключа нет.
- `netlify.toml`: publish ".", functions "netlify/functions".

### Фронт: `index.html` (по-прежнему один файл)
- `askVLM` ходит на `/api/vlm` (same-origin), без Authorization; `AbortSignal.timeout(15000)` —
  зависший запрос больше не останавливает цикл навсегда.
- Убраны getKey()/prompt()/localStorage.

### Голосовая цель (STT)
- Кнопка «🎤 Цель голосом»: `webkitSpeechRecognition`, ru-RU, одна фраза.
- Распознанная фраза целиком становится целью (VLM понимает свободную формулировку
  «я хочу дойти до бутылки воды»), поле цели обновляется, голос подтверждает
  «Веду к: …», навигация стартует автоматически, история команд сбрасывается.
- Пока идёт распознавание — TTS глушится и подавляется (иначе микрофон слышит телефон).
- Нет SpeechRecognition (iOS старый) → голосовая кнопка озвучивает недоступность,
  остаётся текстовое поле.

### Режим «что впереди» (цель пуста)
- Тот же цикл, другой системный промпт: «опиши одной фразой (≤10 слов), что перед
  человеком; препятствия — в первую очередь». Та же JSON-схема, `arrived` игнорируется.
- Задали цель (голосом или текстом) → следующий тик уже ведёт к цели.

### Демо-сценарий
Старт камеры → «Навигация» (описывает комнату) → «🎤 хочу дойти до бутылки воды» →
ведёт голосом и вибрацией → «Цель достигнута».

## Надёжность (полевые риски)
- Кадр при `videoWidth == 0` (метаданные не загружены) — тик пропускается, ретрай.
- `track.onended` (камеру забрало другое приложение) — голосовое предупреждение.
- `cam.play()` на visibilitychange — страховка от замороженного видео после сворачивания.
- cancel()+speak() подряд дропает утерансы на Android — speak с микрозадержкой
  после cancel, кроме первого утеранса из user gesture (требование iOS).
- Ошибки API (401/402/429/500/таймаут) — тик пропускается, ошибка в judge-лог,
  цикл живёт.

## Вне скоупа
Карты, GPS, распознавание лиц, настройки, аккаунты, история сессий, красивый UI.
